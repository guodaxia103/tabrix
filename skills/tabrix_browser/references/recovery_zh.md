# Tabrix 恢复层

## 目标

当 Tabrix 无法立刻工作时，先恢复链路，再继续当前任务。

## 常见问题

### 连接失败

优先检查：

1. Tabrix 服务是否运行
2. 扩展是否已加载
3. 浏览器是否已连接 Native Host
4. 远程模式下 Token 是否有效

### 工具超时

- 缩小操作范围
- 减少大页面一次性读取
- 避免长截图或过重组合操作

### 找不到元素

- 重新读取页面
- 先确认页面是否已完成更新
- 优先改用文案或语义定位

## 与用户沟通

始终用以下顺序：

1. 说明问题
2. 说明可能原因
3. 说明下一步建议

不要直接抛出原始堆栈作为最终答复。
