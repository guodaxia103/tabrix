# Tabrix 平台支持说明

本文档说明 Tabrix 当前对各平台的公开支持口径。

## 当前支持定位

| 平台 | 当前定位 |
| --- | --- |
| Windows | 主验证平台 |
| macOS | 原理上支持，但仍建议补真实机器验证 |
| Ubuntu / Linux | 原理上支持，但仍建议补真实机器验证 |

## 对外应该怎么表达

- Windows 是当前最强的验证基线
- macOS / Ubuntu 不是“不支持”，但对外承诺应更克制
- 不要把“代码里有分支”直接说成“生产级完全验证”

## 各平台说明

### Windows

- 当前安装、扩展连接、诊断恢复、发布验收的主验证基线

### macOS

- 浏览器路径探测和服务链路代码分支已具备
- 真实机器上的启动、自启动与用户会话行为仍值得继续验证

### Ubuntu / Linux

- 浏览器探测和服务链路代码分支已具备
- 对扩展连接与桌面行为仍建议继续做真实机器验证

## 公开表达规则

如果某个版本没有完成真实机器验证，就不要把 macOS / Linux 直接表述为“已经完整生产验证”。
